About Task Schedule

Task Schedule is a lightweight, flexible web application designed for managing recurring tasks and schedules. Built for simplicity and ease of use, it runs on your local network or in the cloud, making it accessible from any device.

Technologies Used

Backend

  • Python - Primary programming language for the application logic
  • Flask - Lightweight web framework that powers the application routes and request handling
  • SQLite - Embedded database engine for storing tasks, schedules, and user data with zero configuration

Frontend

  • HTML - Semantic markup for structuring the application interface
  • CSS - Custom stylesheets for responsive design and print-optimized layouts
  • JavaScript - Client-side interactivity for dynamic form fields and user interface enhancements
  • Jinja2 - Server-side templating engine integrated with Flask for generating dynamic HTML

Cloud Infrastructure

  • Amazon Web Services (AWS) - Cloud computing platform hosting the production deployment
  • Amazon EC2 - Elastic Compute Cloud instances running the Flask application server
  • AWS VPC - Virtual Private Cloud for network isolation and security group configuration
  • Amazon Linux - Operating system running on the EC2 instances

Development & Deployment

  • Git - Distributed version control system for tracking code changes
  • GitHub - Code repository hosting and collaboration platform
  • AWS CLI - Command-line tools for automating AWS resource provisioning
  • Systemd - Service manager ensuring the application runs continuously and restarts on reboot

Key Features

  • Flexible Scheduling - Supports 15 different frequency patterns including daily, weekly, monthly, seasonal, and custom intervals
  • User Management - Multi-user support with task assignment capabilities (everyone or specific users)
  • Multiple Views - Alphabetical task listing, chronological timeline, and custom date range views
  • Print Optimization - Clean, lean print layouts designed for standard printer output
  • Responsive Design - Works seamlessly on desktop computers, tablets, and mobile phones
  • Network Accessible - Run on local network or deploy to the cloud for internet access

Schedule Patterns Supported

  1. Interval-based: Every X days/weeks/months starting from a specific date
  2. Weekly on specific day (e.g., every Tuesday)
  3. Ordinal day of month (e.g., first Saturday of each month)
  4. Ordinal day of even/odd months (e.g., first Saturday of even months)
  5. Monthly on specific date (e.g., 15th of each month)
  6. First or last day of each month
  7. First or last day of every X months
  8. X times per month
  9. Yearly in specific week of specific month
  10. Yearly on specific date (e.g., March 15 every year)
  11. Seasonal tasks (Spring, Summer, Fall, Winter)
  12. One-time tasks on specific dates

Architecture Philosophy

This application embraces simplicity and pragmatism. Rather than adding layers of abstraction and complexity, it uses straightforward Python code, minimal dependencies, and proven technologies. The database schema is designed to be flexible enough to handle complex scheduling patterns while remaining easy to understand and modify.

The lightweight architecture means fast page loads, low resource usage, and easy deployment. The entire application runs efficiently on a single small EC2 instance, making it cost-effective for personal or small team use.
